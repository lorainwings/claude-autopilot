# 变异测试策略 (Mutation Strategies)

## 设计哲学

shell 层面的变异测试不追求语义完整性（如 PIT/Stryker 等成熟工具），而是聚焦于**最容易引入 bug 的 6 类常见模式**，覆盖率换简洁性与可控性。

## 6 类 Mutator

| ID | 原始 | 变异后 | 检测目标 |
|----|------|--------|---------|
| M1 | `==` | `!=` | 条件颠倒 — 测试是否覆盖正反两路径 |
| M2 | `>` | `<` | 阈值颠倒 — 测试是否覆盖边界 |
| M3 | `return 0` | `return 1` | 函数成功路径 — 测试是否检验返回值 |
| M4 | `exit 0` | `exit 1` | 脚本出口码 — 测试是否检验退出 |
| M5 | `-eq` | `-ne` | 数值比较 — 测试是否覆盖等号语义 |
| M6 | `"true"` | `"false"` | 字符串字面量 — 测试是否检验配置值 |

## 选择规则

- 每文件**只应用首条匹配**的 mutator，避免组合爆炸
- 跳过注释行（`#` 开头）
- 不修改源文件以外的 artifact

## 安全保证

1. **干净 git 工作区前置检查**：`git status --porcelain` 必须为空
2. **变异前后双重 diff 校验**：变异前确认无残留，变异后 `git checkout --` 强制恢复
3. **trap INT/TERM 自动恢复**：用户中断时不会留下脏文件
4. **超时强制 kill**：避免无限循环导致测试套件挂死

## 已知局限

- 仅匹配字面 pattern，不解析 AST
- shell 没有"unused variable" / "off-by-one" 等高级 mutator
- 无法变异函数签名 / 控制流结构

## 何时不适用

- 目标脚本无对应 test → 全部 survivor，无信号
- test 本身依赖外部服务 / 网络 → timeout 假阳性
- 集成测试场景 → 变异点过多，单 mutator 不具代表性
