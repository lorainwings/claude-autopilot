2. **工作目录:** 测试需跨越 `plugins/spec-autopilot` (底层引擎) 与 `vibe-workflow-app` (前端界面) 两个工作区。

## 🛠️ 分发任务清单 (Task Delegation)

### [Agent 1] 任务一：全模式稳定性与链路闭环测试 (Stability & State Control)

- **对应目标:** 稳定性与流程精准度。
- **执行动作:** 在 `full`、`lite`、`minimal` 三个模式下分别完整跑通底层插件。
- **审计重点:** 状态机流转是否精准受控？文件写入是否正确无截断？最后的代码合并是否无逻辑冲突？
- **输出:** `docs/reports/stability-audit.md`

### [Agent 2] 任务二：Phase 1 质量评审 Benchmark (Requirements Quality)

- **对应目标:** Phase 1 阶段专项评审。
- **执行动作:** 构造 3 个不同复杂度的需求输入。
- **审计重点:** 对“需求分析”、“需求理解”、“调研”、“澄清”四个步骤的产出质量及隐藏约束挖掘能力进行 1-100 分量化打分。
- **输出:** `docs/reports/phase1-benchmark.md`

### [Agent 3] 任务三：全阶段性能与消耗评估 (Performance Metrics)

- **对应目标:** 性能指标制定与评估。
- **执行动作:** 追踪所有模式、阶段的 Token 和时间消耗。
- **审计重点:** 量化“阶段耗时 (Latency)”、“Token 消耗效率 (Token ROI)”以及“无人工干预成功率”，精准定位性能杀手。
- **输出:** `docs/reports/performance-benchmark.md`

### [Agent 4] 任务四：Vibe Coding 顶级竞品综合对比 (Competitive Analysis)

- **对应目标:** 社区热门插件优劣势比对。
- **执行动作:** 对比 `BMAD-method`, `obra/superpowers`, `oh-my-claudecode`, `everything-claude-code`，以及 `Cline`, `Aider`, `OpenHands` 等。
- **审计重点:** 标注当前插件缺失的能力。提取竞品优秀机制，制定将其他插件优势（如会话持久化、防跳步、长上下文检索）扩展到当前插件的详细计划。
- **输出:** `docs/reports/competitive-analysis.md`

### [Agent 5] 任务五：全局规约与记忆遵守度审查 (Compliance & Rules Adherence)

- **对应目标:** 代码生成规约测试。
- **执行动作:** 专项抽查 AI 生成代码及 TDD 流程。
- **审计重点:** 是否严格遵守了项目 `rules` 和 `CLAUDE.md` 的全局记忆？是否发生幻觉、生成 `// TODO` 占位符或重复造轮子？
- **输出:** `docs/reports/compliance-audit.md`

### [Agent 6] 任务六：GUI 交互与双向反控体验审查 (GUI & HITL Interactive Audit)

- **对应目标:** 可视化大盘与底层引擎的协同能力。
- **执行动作:** 启动 `autopilot-server` 和前端 `vibe-workflow-app`，通过浏览器界面观测底层引擎的运行。
- **审计重点:**
  1. **数据渲染同步率:** `PhaseTimeline` (进度条) 和 `ParallelKanban` (并发看板) 是否与底层 JSONL 事件流毫秒级同步？
  2. **终端保真度:** 内嵌的 XTerm.js 虚拟终端是否无损还原了包含 ANSI 颜色的 AI 思考流？
  3. **双向反控验证 (核心):** 故意触发一次 `GateBlock` (如代码覆盖率不足)，测试在 GUI 点击 `Override (强制通过)` 或 `Retry` 按钮后，底层引擎是否能瞬间解除挂起并正确响应指令。
- **输出:** `docs/reports/gui-interaction-audit.md`

### [Agent 7] 任务七：全链路整体性真实业务仿真测试 (Holistic Real-World Simulation)

- **对应目标:** 多维度、多角度的综合集成演练。
- **执行动作:** 在开启 GUI 监控的状态下，驱动插件完整开发以下两个高难场景：
  - **场景 A:** 编排开发企业级 `ConfigCenter` 的分布式客户端拉取模块（考核并发与容灾）。
  - **场景 B:** 开发 `aliyun-oss-uploader` 高性能命令行工具（考核底层 IO 与架构扩展）。
- **审计重点:** 在真实压迫下，前 6 个维度的能力（从代码质量到 GUI 监控不卡顿）是否能完美协同？输出最终的专业性审查结论。
- **输出:** `docs/reports/holistic-simulation-benchmark.md`
