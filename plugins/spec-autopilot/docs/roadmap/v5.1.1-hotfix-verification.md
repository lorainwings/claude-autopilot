# Role: Lead QA Engineer & Regression Auditor (首席 QA 工程师 & 回归测试审计员)

# Objective: 针对 v5.1.1 版本的 5 项核心 P0/P1 级热修复（Hotfix）进行专项代码级审查与逻辑闭环验证。出具确定性的验收报告，确保所有漏洞已被彻底封堵，且未引入新的回归缺陷（Regression）

## ⚠️ 核心审计纪律 (Rules of Engagement)

1. **证据驱动 (Evidence-Based):** 你的每一个验证结论（PASS/FAIL）必须附带具体的代码行号和逻辑片段作为证据。
2. **零信任推理 (Zero-Trust Reasoning):** 不要只看代码是否“被修改了”，而要在脑内推演边界情况（如并发读写、断线重发、空对象）下，修复代码是否依然坚固。
3. **只读审计 (Read-Only Mode):** 除非发现修复代码存在极其低级的致命语法错误（如括号未闭合），否则本次任务仅做审计，不修改任何业务代码。

## 🛠️ 专项回归测试任务流 (Hotfix Verification Pipeline)

### [Test 1] 验证双向反控路径对齐 (DC-PATH)

- **动作:** 交叉比对 `scripts/autopilot-server.ts` 和 `scripts/poll-gate-decision.sh`。
- **审计标准:**
  1. 服务端写入的 `decision.json` 的绝对路径，是否与引擎侧轮询的绝对路径 **100% 字符串等价**？
  2. 服务端在多会话并发时，能否正确解析到当前 `change_dir`？

### [Test 2] 验证 IN_PHASE5 误判拦截修复 (D-01)

- **动作:** 走查 `scripts/unified-write-edit-check.sh` 中的 Phase 检测逻辑。
- **审计标准:**
  1. 脑内推演：当系统处于 full 模式且 `PHASE1_CP` 存在、但 `PHASE3_CP` / `PHASE4_CP` 不存在时，当前脚本的逻辑是否会错误地将 `IN_PHASE5` 设为 `yes`？
  2. 验证修复后的逻辑是否引入了对 `mode` (full/lite/minimal) 的正确感知？

### [Test 3] 验证 Python3 Fail-Closed 机制 (D-05)

- **动作:** 审查 `scripts/post-task-validator.sh` 的 Python 依赖检测模块。
- **审计标准:**
  1. 当强制模拟 `command -v python3` 返回 false 时，脚本是执行了 `exit 0`（漏洞态），还是向 stdout 输出了标准格式的 `{"decision": "block", ...}` 并以错误码退出（安全态）？

### [Test 4] 验证底层竞态锁与语法规范 (D-06, D-03)

- **动作:** 检查 `scripts/_common.sh` 和 `scripts/check-predecessor-checkpoint.sh`。
- **审计标准:**
  1. `next_event_sequence` 函数是否正确引入了 `flock`（或其他等效原子锁）？在高频并发写入时，是否存在死锁风险？
  2. `check-predecessor-checkpoint.sh` 全局作用域中是否已彻底清除了非法的 `local` 变量声明？

### [Test 5] 验证 GUI 内存泄漏与终端批量渲染 (VT-2, SM-1, WS-3)

- **动作:** 静态审查 `vibe-workflow-app/src/store/index.ts` 和 `VirtualTerminal.tsx`。
- **审计标准:**
  1. **状态去重与上限:** `addEvents` 函数中，是否基于 `sequence` 实现了真正的去重（如使用 Map），且在合并后有 `.slice(-1000)` 等机制截断数组防止内存溢出？
  2. **批量渲染:** `VirtualTerminal` 是否引入了机制（如追踪 `lastRenderedSequence`）来遍历渲染传入的所有新事件，彻底解决了之前“只渲染快照最后一条”的丢事件 Bug？

### [Task 6] 输出最终验收报告

- **动作:** 汇总以上 5 项测试结果。
- **输出:** 将报告写入 `docs/reports/v5.1.1-hotfix-verification.md`。必须包含“验证结论看板（PASS/FAIL）”以及“代码证据截取”。如果全部 PASS，请在报告末尾明确给出“**V5.1.1 具备进入全链路实测条件**”的绿灯放行令。
