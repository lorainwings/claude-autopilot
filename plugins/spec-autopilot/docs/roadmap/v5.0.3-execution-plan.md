## 🛠️ 核心债务清剿任务流 (V5.1 Debt Eradication Pipeline)

### 🔴 P0 级致命缺陷修复 (Critical Blockers)

#### [Task 1] 补全双向反控：引擎侧 decision.json 轮询 (P0-1)

- **背景:** 目前 GUI 写入 `decision.json` 后，引擎侧 `emit-gate-event.sh` 是 fire-and-forget，导致决策无法闭环。
- **动作:** 修改 `autopilot-gate/SKILL.md` (或相应的 gate 脚本)。在发射 `gate_block` 事件后，必须进入 `while true; sleep 1; read decision.json` 的挂起轮询循环。一旦读取到合法的 `Override`, `Retry` 或 `Fix` 指令，立即退出轮询并执行对应动作。

#### [Task 2] 封堵后台 Agent L2 绕过漏洞 (P0-2)

- **背景:** `run_in_background: true` 的 Agent 会触发 `exit 0`，完全绕过 Pre/PostToolUse 的 L2 门禁。
- **动作:** 1. 修改 `check-predecessor-checkpoint.sh`，移除针对后台任务的无脑 `exit 0`，改为保留轻量级的前驱 Checkpoint 存在性检查。
  2. 修改 `post-task-validator.sh`，确保后台任务在完成时，必须接受 JSON 信封验证和反合理化（Anti-rationalization）验证。

#### [Task 3] 补齐 TDD 阶段隔离 L2 门禁 (P0-3)

- **背景:** TDD 期间 (RED/GREEN) 缺乏 L2 Hook 阻止子 Agent 乱写文件，完全依赖 prompt。
- **动作:** 编写或升级具备“TDD 阶段感知”的 Write/Edit Hook。从上下文或状态文件中读取当前所处 TDD 阶段（RED/GREEN/REFACTOR）。若在 RED 阶段，硬阻断对实现文件的写入；若在 GREEN 阶段，硬阻断对测试文件的修改。

#### [Task 4] Phase 1 崩溃盲区：增加中间 Checkpoint (P0-4)

- **背景:** Phase 1 耗时 10-30 分钟，仅在最后存盘，崩溃会丢失全部用户交互进度。
- **动作:** 修改 Phase 1 流转逻辑。在“三路调研完成”以及“每轮决策 (Decision Round) 完成”后，立即写入中间态的 Checkpoint 文件。

### 🟠 P1 级高危隐患修复 (High Priority Tech Debt)

#### [Task 5] Checkpoint 原子性写入保护 (P1-1)

- **背景:** Checkpoint 写入非原子性，断电或崩溃会留下损坏的 JSON 文件。
- **动作:** 将直接 Write `phase-{N}-{slug}.json` 的行为改为：先写入临时文件 `phase-{N}-{slug}.json.tmp` -> 验证 JSON 格式合法 -> 执行原子的 `mv` 重命名覆盖正式文件。

#### [Task 6] 合并 Write/Edit Hook 突破性能瓶颈 (P1-2)

- **背景:** 每次 Write/Edit 触发 3 个串行的 Hook，累计带来最高 35s 的延迟，导致 Phase 5 成为绝对性能杀手。
- **动作:** 将 `write-edit-constraint-check.sh`、`banned-patterns-check.sh` 和 `assertion-quality-check.sh` 三个脚本合并为一个统一的入口脚本，内部按顺序高效执行这三项检查，大幅降低 fork 开销和延迟。

#### [Task 7] 修复 SA-2 逻辑反转 (P1-3)

- **背景:** `write-edit-constraint-check.sh` 错误地“豁免”了 openspec 路径，导致子 Agent 可以违规修改内部状态。
- **动作:** 修复该 Hook 的逻辑反转。增加上下文识别，如果识别到当前是子 Agent 正在执行写入操作，必须**硬阻断 (exit 1)** 对 `openspec/` 和 `checkpoint` 目录的写入，确保系统状态机的绝对隔离。
