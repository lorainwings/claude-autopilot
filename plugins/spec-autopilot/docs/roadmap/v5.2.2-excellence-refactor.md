## 🛠️ GUI 满分飞升任务流 (The Path to 100/100)

### [Task 1] 遥测引擎点火：TaskProgress 实时脉动 (P0)

- **背景:** 缺乏子任务级别的进度推送，导致 ParallelKanban 无法实时更新。
- **动作:**
  1. **引擎侧:** 在 `scripts/emit-phase-event.sh` 中实现 `task_progress` 事件类型。在 Phase 5 的执行循环中，每完成一个 Task（或在 TDD 的 Red/Green 步之间），发射带有 `task_name`, `status`, `tdd_step` 的进度事件。
  2. **前端侧:** 在 Store 中精确消费此事件。更新 `ParallelKanban.tsx`，确保每个子任务的卡片能随着事件的到来，实时变换颜色或展示进度动画。

### [Task 2] 黑客视觉觉醒：终端 ANSI 高保真着色 (P1)

- **背景:** VirtualTerminal 当前写入的是无色的纯文本，丧失了 CLI 的真实感。
- **动作:**
  1. 在底层 `emit-` 脚本中，为不同的事件注入 ANSI 颜色前缀（例如：Error/Block 是红字 `\x1b[31m`，Phase Start 是绿字 `\x1b[32m`）。
  2. 修复 `VirtualTerminal.tsx` 的配置，确保 `xterm.js` 能够正确解析并渲染这些 ANSI 转义码；设置 `disableStdin: true` 消除用户的错误输入预期。

### [Task 3] 沉浸式反控体验：决策闭环与深度介入 (P1)

- **背景:** 决策缺乏服务端 ACK；`Fix` 动作缺乏具体的指导输入框。
- **动作:**
  1. **ACK 闭环:** 修改 `autopilot-server.ts`，在成功写入 `decision.json` 后，立刻通过 WebSocket 广播一个 `decision_ack` 事件；前端收到后瞬间移除 `GateBlockCard`。
  2. **深度介入:** 在 `GateBlockCard.tsx` 中，当用户选择 `Fix`（修复）动作时，展开一个 `<textarea>` 输入框，允许用户填写 `fix_instructions`，并将其包含在决策 payload 中下发给底层引擎。

### [Task 4] 前端工程底座洗礼：清剿技术债 (P2)

- **背景:** 存在未安装的 Tailwind 类名和 Zustand 渲染性能隐患。
- **动作:**
  1. 彻底解决 Tailwind 问题：要么在 `vibe-workflow-app` 中完整初始化并安装 Tailwind CSS，要么将组件中游离的 Tailwind 类名（如 `bg-gray-900`）全部重写为标准的 CSS/Inline Styles。
  2. 优化 Store：将 `taskProgress` 的数据结构从 `Map` 替换为 `Record<string, TaskProgress>`，以解决 Zustand 默认的浅比较（shallow compare）失效问题，防止无效的组件重渲染。
