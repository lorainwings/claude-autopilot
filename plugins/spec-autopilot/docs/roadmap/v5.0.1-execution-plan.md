# Role: Lead React Architect & Interactive UI Expert (首席 React 架构师 & 交互 UI 专家)

# Objective: 移除 `vibe-workflow-app` 中 Phase 5 并发看板的占位符，将其替换为真实的数据绑定组件。同时，打通 Gate Block（门禁拦截）的“GUI -> 引擎”双向反控链路，完成真正在线的闭环编排

## ⚠️ 核心执行纪律 (Rules of Engagement)

1. **数据驱动 UI (Data-Driven):** Kanban 组件的状态必须完全由 Zustand (或你的 Store) 里的数据衍生，杜绝硬编码的假状态。
2. **优雅反控 (Graceful Control):** 在实现点击按钮发送决策时，必须处理好 loading 状态、防抖 (debounce) 和超时反馈，确保交互的专业感。
3. **样式一致性:** 继续使用 Tailwind CSS，保持与现有大盘界面一致的极简、极客风格 (暗黑主题为佳)。

## 🛠️ 界面点亮与双向控制任务流 (UI Hydration & Control Pipeline)

### [Task 1] 前端：点亮 Phase 5 并发看板 (Hydrate Parallel Kanban)

- **目标:** 替换“并行任务看板将在后续版本中实现”占位符。
- **动作:**
  1. 在 Store 中补全解析 `TaskProgressEvent` (包含 `task_name`, `status`, `tdd_step` 等) 的 Reducer 逻辑。
  2. 完善 `ParallelKanban.tsx` 组件。将其设计为网格布局 (Grid)，每个并发 Domain（如 frontend, backend）展示为一个 Card。
  3. 在 Card 内渲染微型状态器：显示当前正在执行的 TDD 步骤（如 `RED` 🔴, `GREEN` 🟢, `REFACTOR` 🔵），以及 ETA 倒计时。

### [Task 2] 前后端：双向决策反控链路 (Two-Way Gate Decision)

- **目标:** 让 GUI 具备实质性的指挥能力。
- **动作:**
  1. **前端侧:** 完善 `GateBlockCard.tsx`。当用户点击 `Retry`, `Fix` 或 `Override` 时，调用 `WSBridge.sendDecision()` 发送标准的 JSON 格式命令给 `ws://localhost:8765`。
  2. **中继侧 (autopilot-server.ts):** 拦截该 WS 消息，并将其安全地写入 `.autopilot/decision.json`，或者直接通过进程间通信 (IPC) 传递给被阻塞的 `spec-autopilot` 主进程。
  3. **引擎侧:** 确保 `spec-autopilot` 的门禁脚本处于挂起轮询状态（如轮询 `decision.json`），一旦读取到 GUI 发来的合法决策，立即跳出阻塞并执行对应动作。

### [Task 3] 前端：XTerm.js 保真度优化 (Virtual Terminal Polish)

- **目标:** 确保右侧/下方的虚拟终端具备生产级可用性。
- **动作:**
  1. 为 `VirtualTerminal.tsx` 增加 `FitAddon`，确保在浏览器窗口 Resize 时，终端能够完美自适应宽度和高度，不出现文字重叠或溢出。
  2. 确保终端开启暗黑主题 (Theme 配置)，并正确解析从后端传来的所有 ANSI 颜色控制码，保持与原生 Claude Code 一致的色彩体验。

### [Task 4] 联调与测试模拟 (Integration & Mock Play)

- **目标:** 在不启动真实厚重业务项目的情况下验证 UI。
- **动作:** 扩展现有的 `mock-event-emitter.js`。让它能在启动后 10 秒内，模拟发射 3 个并发任务的 `TaskProgressEvent` 填满看板，并在第 15 秒模拟抛出一个 `GateBlock` 事件，供人工点击测试反控链路。
