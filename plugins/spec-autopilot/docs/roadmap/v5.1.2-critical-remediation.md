## 🛠️ 全量修复任务矩阵 (Remediation Matrix)

请系统性地阅读并依次执行以下三大模块的修复任务：

### 模块一：L2 确定性铁壁与 TDD 引擎加固 (Backend & Determinism)

* **任务 1：实现 TDD-5 自动回滚脚本 (报告 C3-1)**
  * **行动**：在 `scripts/` 下新建 `tdd-refactor-rollback.sh`。不要让回滚依赖 AI 意图，必须脚本化。逻辑需包含：用 `git stash` 保存当前非相关的未提交改动 -> 执行核心代码回滚 (`git checkout`) -> `git stash pop` 恢复。
* **任务 2：落地 `min_qa_rounds` 的 L2 硬阻断 (报告 C1-1)**
  * **行动**：修改 `scripts/_config_validator.py`，在 `RANGE_RULES` 中为 `min_qa_rounds` 增加 `(1, 10)` 的范围校验。修改 `scripts/_post_task_validator.py` (Validator 5)，提取 Phase 1 `decisions` 数组的长度，与配置的最低轮数进行硬对比，不达标直接 `block`。
* **任务 3：堵住 Python3 缺失的 fail-open 漏洞 (报告 C2-1)**
  * **行动**：修改 `unified-write-edit-check.sh` CHECK 4，当环境缺失 python3 时，必须实行 fail-closed（阻断并报错），禁止静默放行。

### 模块二：服务端 I/O 与 WebSocket 容错 (Server & I/O)

* **任务 4：修复 WebSocket 静默中断与 I/O 放大 (报告 C4-1 & M4-1)**
  * **行动**：修改 `autopilot-server.ts` 中的 `broadcastNewEvents`。必须将 `JSON.parse(line)` 包裹在 `try-catch` 中，遇到损坏行记录 error 并跳过，**确保 `lastLineCount` 的更新是在安全解析之后或精确计算偏移量，绝不能错乱**。同时，将 `getEventLines()` 从全量读取优化为基于文件流或字节偏移量的增量读取，彻底解决长会话 I/O 退化。

### 模块三：GUI V2 交互反馈与渲染性能 (Frontend & GUI)

* **任务 5：决策发送失败的 UI 强报警 (报告 C5-1)**
  * **行动**：修改 `gui/src/components/GateBlockCard.tsx`。在 `sendDecision` 的 `catch` 块中，不能只做 `console.error`。必须引入 Error 状态，在卡片顶部或按钮区域渲染醒目的红色错误提示（如“决策发送失败，请检查网络连接”），明确告知用户操作未成功。
* **任务 6：高频 Selector 的 Memoization 优化 (报告 M5-2)**
  * **行动**：修改 `gui/src/store/index.ts` 及消费侧组件。针对 `selectPhaseDurations`、`selectTotalElapsedMs`、`selectGateStats` 等纯函数，使用 `useMemo` 或 Zustand 的 `useShallow` 进行缓存包裹，消除 1 秒定时器引发的大量无谓数组遍历和 React 重渲染。
