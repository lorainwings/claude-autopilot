# Role: Product Manager & UX Architect (首席产品经理 & 交互架构师)

# Objective: 全局扫描 `spec-autopilot` 插件的底层编排逻辑（skills/, scripts/）和当前 GUI，逆向提取出一份极其详尽的《全局功能与 UI 重构 PRD》。这份文档将被送入高级 AI UI 生成工具（如 v0, Stitch）中，用于重新设计一套具有顶级“赛博朋克/黑客帝国”质感的自动驾驶大盘

## ⚠️ 提取纪律

1. **不要写代码**: 你的任务是生成 Markdown 格式的需求与规范文档，不要生成任何 React 或 CSS 代码。
2. **穷尽所有状态**: 仔细阅读底层脚本（如 `emit-phase-event.sh`、`SKILL.md`），把底层能暴露的所有数据（阶段名、错误信息、阻塞原因、持续时间、并发状态）全部提取出来，作为 UI 需要展示的字段。
3. **结构化适配 UI 生成器**: 文档结构必须对 v0 / Stitch 极其友好，清晰定义布局、组件和交互。

## 🛠️ 执行任务流

请全局扫描代码库，并生成文件 `docs/UI-Redesign-PRD.md`，文档必须严格包含以下 5 个部分：

### 1. 视觉风格与交互定义 (Design System & Vibe)

- 描述极其酷炫的视觉基调（例如：Cyberpunk, Dark Hacker Terminal, Deep Space IDE）。
- 定义主色调（背景、边框）和语义色（Red=阻断, Green=通过, Amber=警告/人工介入）。
- 动效要求（例如：终端文字的打字机效果、数据流动的呼吸灯效果、阻断时的全屏脉冲闪烁）。

### 2. 全局布局架构 (Global Layout Structure)

- 为大屏监控设计一个现代化的仪表盘布局（例如：左侧栏纵向 Timeline、中上部并发 Kanban、中右侧环形数据统计、下半屏沉浸式 XTerm 终端）。

### 3. 全局生命周期与模块清单 (Comprehensive Feature List)

- **核心流转 (Phase 0 - Phase 7)**: 详细列出所有 8 个阶段的名称、描述，以及它们在 UI 上的呈现方式。
- **微观并发 (Task/Agent 层)**: 提取 Phase 5 实施、Phase 6 测试等阶段的并行任务属性（状态、进度、耗时）。
- **拦截门禁 (Gate System)**: 提取 L1/L2/L3 门禁触发时的属性（拦截原因、门禁得分、修复建议）。

### 4. 核心 UI 组件蓝图 (Core Component Blueprints)

详细描述每个核心组件应当长什么样、包含什么数据：

- `<PhaseTimeline>`: 带有流光效果的当前阶段指示器。
- `<ParallelKanban>`: 类似赛车仪表盘的并发子任务监控。
- `<InterventionModal>` (拦截阻断弹窗): 必须包含 Override 按钮、Retry 按钮和 `fix_instructions` 的多行输入框。
- `<TelemetryDashboard>`: 统计 Token 消耗、Hook 耗时等极客数据。
- `<HackerTerminal>`: 支持 ANSI 高亮的实时日志流黑框。

### 5. Mock 数据结构 (State Interface for UI Gen)

- 提供一份完整的、复杂的 JSON Mock 数据样例。包含所有的 events、当前的 phase、多个并发 task 等。这是为了让 UI 生成器能直接拿去渲染出有血有肉的界面。
